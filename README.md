# AI Email Automation Agent

Локальный backend-сервис для автоматического анализа входящих бизнес-писем. Сервис принимает тему и текст письма, передаёт их локальной LLM через Ollama и возвращает структурированный JSON: категорию, приоритет, краткое содержание, черновик ответа и рекомендуемое действие.

Проект не использует платные или облачные AI API. Данные обрабатываются локально.

## Возможности

- анализ темы и текста письма локальной моделью Ollama;
- структурированный JSON с категорией, приоритетом, бюджетом и черновиком ответа;
- автоматическое сохранение истории в SQLite;
- получение истории через FastAPI;
- Telegram-интерфейс с меню, статистикой и карточками обращений;
- экспорт истории в Excel;
- отправка действий для Google Calendar в n8n webhook.

## Быстрый старт после клонирования

1. Клонируйте репозиторий и перейдите в его папку:

```powershell
git clone <URL_РЕПОЗИТОРИЯ>
cd ai-gmail-agent
```

Если проект скачан ZIP-архивом, распакуйте его и откройте PowerShell в папке проекта.

2. Создайте виртуальное окружение и установите зависимости:

```powershell
python -m venv venv
.\venv\Scripts\Activate.ps1
python -m pip install -r requirements.txt
```

3. Создайте локальный файл настроек:

```powershell
Copy-Item .env.example .env
```

Откройте `.env` и вставьте собственный Telegram-токен. Файл `.env.example` должен оставаться шаблоном без реальных секретов.

4. Установите Ollama и скачайте локальную модель.

Откройте официальную страницу [Ollama для Windows](https://ollama.com/download/windows), скачайте `OllamaSetup.exe` и выполните установку. После запуска Ollama работает в фоне, добавляет команду `ollama` в PowerShell и поднимает локальный API на `http://localhost:11434`.

Закройте и заново откройте PowerShell, затем проверьте установку:

```powershell
ollama --version
```

Выберите подходящую модель в [библиотеке Ollama](https://ollama.com/library). Для старта можно использовать небольшую мультиязычную модель `qwen2.5:3b`:

```powershell
ollama pull qwen2.5:3b
```

Команда `pull` скачивает модель на компьютер. Размер и требования зависят от выбранной модели, поэтому выбирайте вариант под объём оперативной памяти и свободного места.

Проверьте список установленных моделей:

```powershell
ollama list
```

При желании протестируйте модель напрямую:

```powershell
ollama run qwen2.5:3b
```

После проверки завершите диалог командой `/bye`. Если выбрали другую модель, скачайте её аналогично и укажите точное имя в `.env`:

```env
OLLAMA_MODEL=имя_вашей_модели
```

Рекомендуется выбирать instruct-модель, которая уверенно следует инструкциям и умеет возвращать JSON. Значение `OLLAMA_MODEL` должно точно совпадать с именем, которое показывает `ollama list`.

5. Запустите backend:

```powershell
uvicorn app.main:app --reload
```

API будет доступен по адресу `http://localhost:8000`, Swagger — по адресу `http://localhost:8000/docs`.

6. Для запуска Telegram-бота откройте второй терминал:

```powershell
.\venv\Scripts\Activate.ps1
python run_bot.py
```

## Использование

Интерактивная документация Swagger:

http://localhost:8000/docs

Откройте `POST /analyze-email`, нажмите **Try it out** и передайте, например:

```json
{
  "subject": "Нужен Telegram-бот для обработки заявок",
  "body": "Здравствуйте, меня зовут Иван. Мы из компании Ромашка. Хотим заказать Telegram-бота для обработки заявок с сайта. Бюджет примерно 100000 рублей, желательно запустить за 2 недели."
}
```

Пример успешного ответа:

```json
{
  "id": 1,
  "category": "sales_lead",
  "priority": "high",
  "client_name": "Иван",
  "company": "Ромашка",
  "summary": "Компания Ромашка хочет заказать Telegram-бота для обработки заявок с сайта. Бюджет — около 100000 рублей, срок запуска — две недели.",
  "budget": "100000 рублей",
  "deadline": "2 недели",
  "needs_reply": true,
  "reply_draft": "Здравствуйте, Иван! Спасибо за обращение. Готовы обсудить требования к боту и предложить план реализации в указанные сроки.",
  "tags": ["telegram", "бот", "лид"],
  "recommended_action": "Связаться с Иваном и назначить короткий созвон для уточнения требований."
}
```

Фактический текст анализа может отличаться, но структура ответа остаётся той же. Если Ollama недоступна, модель не установлена, истёк таймаут или модель вернула некорректный JSON, API вернёт понятную ошибку с HTTP-кодом `502`, `503` или `504`.

## Эндпоинты

- `GET /` — информация о сервисе.
- `GET /health` — проверка работы backend-сервиса.
- `POST /analyze-email` — анализ письма с помощью Ollama.
- `GET /history?limit=10` — последние сохранённые обращения.
- `GET /history/{id}` — одно обращение по идентификатору.

## Настройки окружения

| Переменная | Значение по умолчанию | Назначение |
|---|---|---|
| `OLLAMA_URL` | `http://localhost:11434/api/generate` | Endpoint генерации Ollama |
| `OLLAMA_MODEL` | `qwen2.5:3b` | Имя выбранной и заранее скачанной модели Ollama |
| `TELEGRAM_BOT_TOKEN` | отсутствует | Токен собственного бота от BotFather |
| `N8N_CALENDAR_WEBHOOK_URL` | локальный n8n webhook | Передача запроса на создание события |
| `N8N_EXPORT_WEBHOOK_URL` | локальный n8n webhook | Зарезервированный webhook экспорта |

## Локальная история обработанных писем

После успешного вызова `POST /analyze-email` исходное письмо и результат анализа автоматически сохраняются в локальную SQLite-базу. В ответ анализа добавляется поле `id` сохранённой записи.

Файл базы данных создаётся при запуске приложения в корне проекта:

```text
email_agent.db
```

Для работы с историей доступны два endpoint:

- `GET /history?limit=10` — последние обработанные письма, отсортированные от новых к старым. Допустимый `limit`: от 1 до 100.
- `GET /history/{id}` — одна запись по её идентификатору. Для отсутствующей записи API возвращает `404`.

Поля `tags` хранятся в SQLite как JSON-строка, а через API возвращаются как массив строк. Управлять схемой базы вручную не нужно: таблица `email_analyses` создаётся автоматически при старте FastAPI.

## Telegram bot

Telegram-бот служит локальным интерфейсом управления CRM: показывает последние обращения и статистику, формирует Excel-таблицу и передаёт запросы на работу с календарём в n8n. Логика Gmail и Google Calendar в Python не реализуется.

### Настройка и запуск

1. Создайте нового бота через [@BotFather](https://t.me/BotFather) и получите токен.

2. Скопируйте `.env.example` в `.env` и укажите токен и адрес webhook n8n:

```env
TELEGRAM_BOT_TOKEN=ваш_токен
N8N_CALENDAR_WEBHOOK_URL=http://localhost:5678/webhook/add-calendar-event
N8N_EXPORT_WEBHOOK_URL=http://localhost:5678/webhook/export
```

3. Установите зависимости:

```powershell
pip install -r requirements.txt
```

4. Запустите backend:

```powershell
uvicorn app.main:app --reload
```

5. В другом терминале активируйте то же виртуальное окружение и запустите бота:

```powershell
.\venv\Scripts\Activate.ps1
python run_bot.py
```

### Возможности бота

- `📊 Таблица обращений` — формирует и отправляет `.xlsx` из последних 1000 записей. Файлы сохраняются локально в папке `exports/`.
- `📋 Последние обращения` — показывает пять последних обращений и действия для каждого.
- `📈 Статистика` — выводит число обращений по категориям и приоритетам.
- `📅 Добавить в календарь` — отправляет данные обращения в `N8N_CALENDAR_WEBHOOK_URL`; дальнейшую работу выполняет n8n.
- `✉️ Черновик ответа` — показывает подготовленный Ollama текст ответа.
- `✅ Закрыть` — пока подтверждает действие без изменения записи.

Если в текущей базе отсутствует email отправителя, в Excel будет указано `не указан`.

## Структура проекта

```text
app/
├── export/             # генерация Excel-файлов
├── telegram_bot/       # Telegram-интерфейс
├── crud.py             # операции с историей
├── database.py         # SQLite и создание таблицы
├── llm_client.py       # HTTP-клиент Ollama
├── main.py             # FastAPI-приложение
├── prompts.py          # промт анализа
└── schemas.py          # Pydantic-схемы
examples/               # пример входящего письма
.env.example            # безопасный шаблон настроек
requirements.txt        # Python-зависимости
run_bot.py              # запуск Telegram-бота
```

Локальные `.env`, `venv/`, `email_agent.db`, `exports/` и Python-кэши исключены через `.gitignore` и не должны попадать в репозиторий.

## Безопасность

- Никогда не сохраняйте реальный Telegram-токен в `.env.example`, README или Git.
- Если токен был опубликован, отзовите его через `@BotFather` и создайте новый.
- Не публикуйте `email_agent.db`: база может содержать тексты писем и персональные данные.
